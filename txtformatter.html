TextSweeper

Duration: 1 Month (In Progress)

Project Type: CSE 360 Group Project

Team Members: Ben Danek, Rachel White, & me

My Role: UX Researcher & Developer

Tools: Java, Java Swing, Github

Skills: Product Development, Teamwork, Time Management, UX Writing


This project is intended to format text based on flags embedded in the unformatted text. The current repository contains the basis for a GUI, there is no event handling so far. The point of this is to be have a model we can use for a user guide which is due much further in advance than the final product. The class this is for is supposed to develop robust customer interaction practices, and understanding of how to work on a software team. This page will be updated as the project progresses.

Problem Statement

Our customer is requesting a tool that takes in an input text file and outputs a formatted output file.

Goal

Because the problem statement is extremely vague, we've furthur questioned the customer for more information: How is the format determined of the output file? We've learned that the customer does not have a preference of the format and that the input text file is a raw text file with commands embedded into it. The customer has given a set of commands that need to be "translated" and has left the rest of the product determinites to our team. Thus, our goal for this project is to develop a program that reads in an input file, processes the commands, and generates the output file.

Research

For research, Ben, Rachel, and I each searched similar products on the market. We've learened that there are various text formatters that are offered online. Most of these products offer a screen that allows the user to see the text prior to downloading, which is something we included in our design as well. As our customer has requested an offline product, we decided to create a program that would launch as a program from a user's desktop which we thought to use Github for.

Challenges

  • How to account for user errors?
  • Will certain commands will override the other?
  • Is there a max character limit?
  • How will a user input the commands?
  • If we decide to use Github, are there pre-requirements for the program run successfully?

Task Development

For the development portion of the product, we broke down our project into two steps: 1- creating the front end, 2- adding the functionality (back-end). As Java Swing was something I've previously used in a class, we decided to use Java Swing to implement a GUI (Graphical User Interface) and also adding functionalty with Java. In an overview, our product is a program that takes in a user-selected text file, "translates" the raw text from the given commands, and genreated the output file. Now that we've tackeled the design challenges, it was time to create a mockup.

Design Sketches & Ideas

Using our research, we came up with 2 different designs for our product. After clarifying with the customer, it was confirmed that the text would be embedded with the commands; thus, scrapping the need for a command line in the program. GUI Prototype

We made a prototype with Java that we conducted our usability testing with. Currently, our mockup is our design translated into Java Swing and does not contain any back-end functionality as of yet. We focused on creating features that would work for our users in 3 steps.

Task 1: Add the text file into the program

Initiate the process of the program by clicking on the ‘Open’ button and selecting a text (.txt) file from the computer directory. The now ingested text should be visible in the main textbox of the program

Task 2: Format the text

Clicking format will parse the source text for formatting flags, and reformat the source text to match the flags. The intended role of the user is to include the formatting flags as desired to describe where and how a formatting change belongs. Use the scroll bar down this section to see the entire text.

Task 3: Display Errors (if any)

If any errors are seen, the program will display the error condition in the ‘Error’ box on the bottom section in red text.

Task 4: View text in sample window

Once the desired results are attained, click on the ‘Save’ button to generate a file that contains the results labeled as Output.txt in the same directory as the source file.

Error Conditions

While using the TextSweeper, general conditions such as unrecognized commands will be handled by the program; however, all actions marked as unrecognized by the program will trigger an error message printed on the screen:


  • Invalid Command: Command not recognized as one of the possible commands.
  • Unable to open input file: Input file selected did not contain a .txt extension.
